在Git中，查看服务器上分支的更新记录可以通过几种不同的方式实现。以下是一些常用的命令：

查看远程分支列表：
使用 git fetch 命令来获取最新的远程分支信息，但不合并它们。然后，你可以使用 git branch -r 来查看远程分支的列表。

git fetch --all
git branch -r
查看特定远程分支的日志：
如果你想要查看特定远程分支的更新记录，可以使用 git log 命令加上远程分支的名称。

git log origin/branch-name
其中 origin 是默认的远程仓库名称，branch-name 是你想要查看的分支名称。

查看所有远程分支的更新记录：
如果你想要查看所有远程分支的更新记录，可以使用以下命令：

git fetch --all
git log --oneline --graph --decorate --all
这将显示一个简洁的图形化日志，包括所有的分支。

查看远程分支相对于本地分支的差异：
如果你想要查看远程分支相对于你的本地分支有哪些差异，可以使用 git diff 命令。

git diff HEAD origin/branch-name
这将显示本地 HEAD（通常是当前分支的最新提交）和远程分支之间的差异。

查看远程分支的最新提交：
如果你只对远程分支的最新提交感兴趣，可以使用以下命令：

git log -1 origin/branch-name
这将只显示远程分支的最新一个提交记录。

请根据你的具体需求选择合适的命令。如果你已经有一个本地仓库并且想要同步最新的远程分支信息，通常第一步是使用 git fetch 来更新本地的远程跟踪分支，然后再查看日志或其他信息。
